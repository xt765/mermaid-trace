# 更新日志 (Changelog)

本项目的所有重要更改都将记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
并且本项目遵守 [Semantic Versioning](https://semver.org/lang/zh-CN/)（语义化版本控制）。

## [0.5.2] - 2026-01-27

### 新增
- **列表/元组项合并**: 自动合并列表和元组中连续的相同项（例如 `['a', 'a', 'a']` -> `['a' x 3]`），在处理包含大量重复对象的集合时显著减少图表中的视觉噪音。

### 改进
- **测试覆盖率**: 通过为 CLI、装饰器和处理器添加全面的边缘情况测试用例，将测试覆盖率提升至 **96.22%**。
- **鲁棒性**: 通过修复 Mock 处理逻辑并确保正确的资源清理，改进了 CLI 错误处理和热重载的稳定性。
- **文档体系**: 同步了所有中英文文档，并为所有核心模块更新了详尽的源代码中文注释。

## [0.5.1] - 2026-01-27

### 新增
- **对象显示优化**: 自动简化 Mermaid 图表中的 Python 对象表示（例如 `<__main__.AuthService object at 0x...>` -> `<AuthService>`），提升图表整洁度并消除无意义的内存地址信息。

### 改进
- **文档体系**: 全面更新了中英文文档集，包括用户指南、API 参考以及详尽的源代码中文注释，确保文档与当前 0.5.x 版本代码同步。

## [0.5.0] - 2026-01-27

### 新增
- **智能折叠**: 自动折叠循环中的重复高频调用，将其合并为带计数器的单个箭头（如 `func (x10)`），防止图表膨胀。
- **自动化插桩**: 新增 `@trace_class` 装饰器，支持一键追踪类中的所有公共方法。
- **第三方库补丁**: 新增 `patch_object` 工具，支持在不修改源码的情况下追踪外部库方法（如 `requests.get`）。
- **全局配置系统**: 引入 `MermaidConfig` 系统，支持通过代码或环境变量全局控制参数捕获、字符串长度限制和递归深度。
- **完整堆栈捕获**: 异常发生时自动捕获完整的 Python 堆栈追踪，并在 Mermaid 图表中通过 Note 展示，极大简化了调试过程。

### 变更
- **`configure_flow` API**: 扩展支持 `level`, `config_overrides` 和 `queue_size` 参数，初始化更灵活。
- **增强型 FastAPI 集成**: 中间件现在可以捕获并记录未处理异常的完整堆栈信息。

### 改进
- **文档完善**: 为所有核心模块补充了详尽的源代码中文注释文档。
- **测试覆盖率**: 保持 >93% 的覆盖率，并新增了针对折叠逻辑和配置系统的测试用例。

## [0.4.2] - 2026-01-27

### 修复
- **延迟加载失效**: 修复了 `MermaidFileHandler` 忽略 `delay=True` 参数的问题。现在仅在产生第一条日志时创建文件，避免生成不必要的空文件。
- **命名冲突**: 修复了 `MermaidFormatter._sanitize` 中的命名冲突问题（例如 `User A` 与 `User-A`），通过自动冲突解决策略确保 Mermaid ID 的唯一性。

### 改进
- **代码清理**: 移除了 `Event` 和 `FlowEvent` 中冗余的 Getter 方法，改用更符合 Python 风格的属性访问。
- **职责解耦**: 将 Mermaid 头部生成逻辑从 `MermaidFileHandler` 移动到 `MermaidFormatter.get_header()`，实现了关注点分离。

## [0.4.1] - 2026-01-26

### 新增
- **抽象事件模型**: 引入了 `Event` 抽象基类和 `BaseFormatter` 接口，以提高扩展性并支持多种输出格式。
- **增强型异步处理器**: 添加了队列大小限制 (`queue_size=1000`) 和丢弃策略，以防止高流量场景下的内存溢出。
- **改进的异常处理**: 增强了异常日志记录，包括完整的堆栈跟踪和错误详情。

### 修复
- **上下文丢失问题**: 修复了 `LogContext._get_store()` 方法，在没有上下文时正确初始化 `contextvar`，防止后续 `LookupError` 异常。
- **并发安全性**: 在 `MermaidFileHandler._write_header()` 中添加了线程锁，以防止在 `delay=True` 时写入文件头的竞争条件。

### 改进
- **架构设计**: 通过引入抽象接口解耦组件，减少了 `FlowEvent` 与 Mermaid 特定格式之间的紧密耦合。
- **测试覆盖率**: 通过为新抽象类添加全面的测试用例，将测试覆盖率提高到 90.17%。
- **代码可维护性**: 为所有代码文件添加了详细的英文注释，提高了可读性和开发体验。

## [0.4.0] - 2026-01-26

### 新增
- **异步模式**: 在 `configure_flow` 中引入了 `async_mode=True`，将日志写入操作卸载到后台线程，消除了主应用程序循环中的 I/O 阻塞。
- **数据隐私**: 为 `@trace` 添加了 `capture_args=False`，以防止敏感参数被记录。
- **参数截断**: 为 `@trace` 添加了 `max_arg_length` 和 `max_arg_depth`，以控制记录的数据结构的大小。
- **显式命名**: 为 `@trace` 添加了 `name`（或 `target` 别名）参数，用于显式设置参与者名称，覆盖自动推断。
- **灵活配置**: 更新了 `configure_flow` 以接受自定义 Handler 列表和 `append` 标志，允许更好地与现有日志设置集成。

### 改进
- **测试覆盖率**: 实现了 >90% 的测试覆盖率，拥有覆盖单元、集成和并发场景的全面新测试套件。
- **PyPI 合规性**: 切换到通过 `hatch-vcs` 进行动态版本控制，并改进了包元数据和工件包含。

## [0.2.0] - 2026-01-26

### 重大转型：可视化优先
MermaidTrace 已从通用的日志包装器转型为专门的 **执行流可视化工具**。目标是直接从运行中的 Python 代码生成 Mermaid 时序图。

### 新增
- **`@trace` 装饰器**：自动记录函数调用为时序交互。支持捕获参数和返回值。
- **上下文推断**：使用 `contextvars` 自动检测 `source`（源）参与者，无需手动连线即可实现嵌套调用可视化。
- **Mermaid 处理器**：专门的日志处理器，实时写入 `.mmd` 文件。
- **CLI 工具**：添加了 `mermaid-trace serve <file.mmd>` 命令，可在浏览器中预览生成的图表。
- **FastAPI 中间件**：`MermaidTraceMiddleware` for zero-config HTTP request tracing.
- **数据捕获**：支持在图表中捕获并显示函数参数和返回值。

### 移除
- **旧版后端**：移除了通用的 `StandardLogger` 和 `LoguruLogger` 包装器。
- **旧版集成**：移除了旧的 Flask/Django 集成（由专注于流程可视化的中间件取代）。
- **JSON 格式化器**：移除，改为 Mermaid 格式输出。

### 变更
- **包元数据**：更新了 `pyproject.toml` 的关键字、描述和分类器以反映新重心。
